Cues & Behavioural Design

Presenting hierarchical information using visual properties, position and shape of the information displayed drives focus of application design.
Shape: Simple differentiating shape from other icons that enhance recognisability.
Value:Contrasts in value define the usability and visual cue performance to draw user attention to particular design elements or in the instance of guiding a user through the process.
Colour: to create effective visual identity that allows users to recognise similarities and differences between design elements. Careful crafting of colour theory enhances user visual sense, recognition and understanding.
Position: Delivering correct order of information needs positioning detail to locate the element on smaller screens and be coherent with user’s natural language flow.

Application interface

Use visual properties to group elements and create a clear hierarchy
Provide visual structure and flow on each screen.
Use cohesive, consistent and contextually appropriate imagery.
Integrate style and function comprehensively and purposefully.
Avoid visual noise and clutter.
During the process of asset selection designers should pay close attention to relating intuition with human learning process in order to lower the cognitive hurdles faced by users. Careful selection of design elements (metaphors and idioms) will deliver simple learning, recognisability and more so user (persona) centred design paradigm. Implementation and redesign of the current project shall only endeavour the change if in future more specific implementation of elements fit best for the user.
